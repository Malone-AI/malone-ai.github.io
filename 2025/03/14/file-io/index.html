C语言文件IO常用函数

发表于2025-03-14|更新于2025-03-14|嵌入式
|浏览量:

系统调用IO函数都是围绕着文件描述符fd进行相关操作的

1. fopen打开流

FILE *stream， 文件指针，FILE结构体又被称作流。

  • 参数
    • path：指定文件路径,如：“./test.txt”
    • mode：指定文件的打开方式
  • 返回值
    • 成功，返回指向该文件的文件指针
    • 若失败，返回NULL

mode参数：

模式 说明
“r” 打开文本文件用于读
“w” 创建文本文件用于写，并删除已存在的内容(如果有的话)
“a” 添加；打开或创建文本文件用于在文件末尾写
“rb” 打开二进制文件用于读
“wb” 创建二进制文件用于写，并删除已存在的内容(如果有的话)
“ab” 添加；打开或创建二进制文件用于在文件末尾写
“r+” 打开文本文件用于更新(即读和写)
“w+” 创建文本文件用于更新，并删除已存在的内容(如果有的话)
“a+” 添加；打开或创建文本文件用于更新和在文件末尾写
“rb+” 打开二进制文件用于更新(即读和写)
“r+b” 打开二进制文件用于更新(即读和写)
“wb+” 创建二进制文件用于更新，并删除已存在的内容(如果有的话)
“w+b” 创建二进制文件用于更新，并删除已存在的内容(如果有的话)
“ab+” 添加；打开或创建二进制文件用于更新和在文件末尾写
“a+b” 添加；打开或创建二进制文件用于更新和在文件末尾写

2. fclose关闭流

  • 参数
    • stream：文件指针（流）
  • 返回值
    • 成功，返回0
    • 若失败，返回EOF

在该文件被关闭之前，冲洗缓冲中的输出数据。
缓冲区中的任何输入数据被丢弃。如果标准I/O库已经为该流自动分配了一个缓冲区，则释放此缓冲区。
当一个进程正常终止时（直接调用exit函数，或从main函数返回），则所有带未写缓冲数据的标准IO流都被冲洗，所有打开的标准IO流都被关闭

3. fwrite对打开的流写入

  • 参数
    • ptr：指向要写入数据的指针
    • size：指定写入的每个数据项的字节数，如调用sizeof(char)
    • nmemb：指定写入的数据项的个数
    • stream：要写入的文件流
  • 返回值
    • 返回实际写入的数据项的个数

假设我们有一个包含学生信息的结构体数组，我们想将这些信息写入一个文件中：

  1. 打开文件：使用 fopen 函数以二进制写入模式 ("wb") 打开文件 students.dat。如果文件打开失败，程序会输出错误信息并退出。

  2. 写入数据：使用 fwrite 函数将 students 数组中的数据写入文件。sizeof(Student) 表示每个学生信息的大小，3 表示要写入的学生数量。

  3. 检查写入结果：fwrite 返回成功写入的数据项数量。如果返回值不等于预期的数量（这里是 3），则说明写入过程中出现了错误。

  4. 输出结果：如果一切顺利，程序会输出“数据成功写入文件”。

5. fseek定位流

  • 参数
    • stream：指定重定位文件指针
    • offset：指定偏移量，整数表示正向偏移，负数表示负向偏移
    • whence：指定偏移基准位置
      • 宏SEEK_CUR（当前位置）
      • SEEK_END（文件结尾）
      • SEEK_SET（文件开头）
  • 返回值
    • 成功，返回0
    • 若失败，返回-1，且不改变指针位置。
